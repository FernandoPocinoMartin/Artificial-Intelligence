Normalización de datos¶

Algunos algoritmos (como las SVMs) sufren cuando los rangos de valores entre variables difieren significativamente. Es decir, que una variable se mueva en el intervalo [10,20] y otra en el intervalo [1000, 2000], hará que el algoritmo se pierda.

Para evitar esto, una buena práctica es normalizar todos los valores, haciendo que estén siempre en un intervalo concreto, de esta manera evitamos estos posibles efectos negativos.

El código que aparece a continuación sirve para normalizar los datos utilizando la técnica de StandardScaler, la cual transforma las variables para que tengan media cero y varianza unitaria. Asimismo, podemos observar que la normalización de los datos es útil para evitar que algunas variables tengan más peso en el modelo debido a su escala, y también para mejorar la estabilidad y convergencia de algunos algoritmos de aprendizaje automático.

La SVM Lineal tiene un parámetro C:

  • ¿Cómo varía el acierto (score) cuando aumentamos C?

El parámetro C en una SVM Lineal controla el grado de error que se permite en la clasificación de los datos de entrenamiento. Un valor pequeño de C permitirá un mayor error de clasificación en el conjunto de entrenamiento, mientras que un valor grande de C intentará clasificar cada punto correctamente en el conjunto de entrenamiento.

  • ¿Qué sucede cuando disminuye?

Cuando disminuimos el valor de C, el modelo puede tener una mayor capacidad de generalización y puede tener un mejor rendimiento en el conjunto de prueba, pero puede que tenga un bajo rendimiento en el conjunto de entrenamiento.

  • ¿En qué escala se aprecian los cambios: Escala logarítmica o escala lineal?

Se aprecian mejor los cambios en una escala logarítmica que en una escala lineal, ya que el valor de C se multiplica por una constante en cada iteración.

El código que aparece a continuación sirve para crear y entrenar un modelo de Máquinas de Vectores de Soporte (SVM) con un kernel lineal, utilizando el conjunto de datos de iris como ejemplo. Los SVM son un tipo de modelo de aprendizaje supervisado utilizado para clasificación y regresión. En este caso, el modelo SVM se entrena con los datos de entrenamiento (X_train, y_train) y luego se utiliza para predecir las etiquetas (y_pred) de los datos de prueba (X_test), basándose en el conocimiento previo aprendido durante el entrenamiento. Lo que nos transmite este código es predecir la especie de la flor de iris a partir de sus características (ancho y largo del sépalo y del pétalo) que se encuentran en el conjunto de datos.

In [13]:
# Creamos una SVM linear (pista: usaremos SVC y no LinearSVC)
from sklearn import datasets
from sklearn.model_selection import train_test_split
from sklearn.svm import SVC

# Cargar datos
iris = datasets.load_iris()
X = iris.data
y = iris.target

# Dividir datos en conjuntos de entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=0)

# Crear modelo SVM lineal
clf = SVC(kernel='linear')

# Entrenar el modelo
clf.fit(X_train, y_train)

# Predecir etiquetas para los datos de prueba
y_pred = clf.predict(X_test)

Optimización de Parámetros¶

Cada modelo posee un determinado número de parámetros que proporcionarán mejor/peor rendimiento en función de los datos disponibles. El enfoque es siempre, utilizando los datos de entrenamiento y validación, encontrar la mejor combinación y verificar su rendimiento con los datos de test.

Aunque existen varios mecanismos, vamos a ver cómo se haría únicamente con uno (RandomizedSearchCV)

El código que aparece a continuación sirve para optimizar los parámetros de un modelo de clasificación SVM (Support Vector Machine) utilizando la técnica de búsqueda aleatoria de hiperparámetros mediante la función RandomizedSearchCV de Scikit-Learn. Finalmente, se evalúa el rendimiento del modelo optimizado utilizando los datos de prueba y se imprime el score obtenido.

In [16]:
# Optimizamos los parámetros de un modelo determinado usando RandomizedSearchCV
from sklearn.model_selection import RandomizedSearchCV
from sklearn.svm import SVC
from scipy.stats import uniform

# Definir una grilla de parámetros
param_grid = {'C': uniform(loc=0, scale=4), 'kernel': ['linear', 'rbf']}

# Crear un modelo SVM
svc = SVC()

# Realizar una búsqueda aleatoria en la grilla de parámetros utilizando RandomizedSearchCV
search = RandomizedSearchCV(svc, param_distributions=param_grid, n_iter=10, random_state=0)
search.fit(X_train, y_train)

# Verificar el rendimiento del modelo utilizando los datos de prueba
score = search.score(X_test, y_test)
print('El score del modelo optimizado es:', score)

tree_f1 = metrics.f1_score(y_test, tree_preds)

tree_precision = metrics.precision_score(y_test, tree_preds)

La curva ROC es una herramienta útil para comparar el rendimiento de diferentes modelos de clasificación. En este caso, hemos comparado un modelo SVM lineal con un modelo de árbol de decisión. Según la curva ROC generada, podemos ver que el modelo SVM tiene un AUC (área bajo la curva) ligeramente mayor que el modelo de árbol de decisión, lo que sugiere que el modelo SVM tiene un mejor rendimiento en términos de tasa de verdaderos positivos y tasa de falsos positivos.

Sin embargo, la curva ROC no nos da información sobre la precisión, el recall o la matriz de confusión. Por lo tanto, debemos evaluar el rendimiento de los modelos usando otras métricas como f1-score, precisión y recall, y examinando la matriz de confusión.

from sklearn.datasets import load_wine
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler, MinMaxScaler, Normalizer
from sklearn.svm import SVC
from sklearn.metrics import accuracy_score

# Cargar el conjunto de datos de vinos
wine = load_wine()

# Dividir los datos en conjuntos de entrenamiento y prueba
X_train, X_test, y_train, y_test = train_test_split(wine.data, wine.target, test_size=0.2, random_state=42)

# Definir los normalizadores
scaler_zscore = StandardScaler()
scaler_minmax = MinMaxScaler()
scaler_l1 = Normalizer(norm='l1')
scaler_l2 = Normalizer(norm='l2')

# Entrenar un modelo SVM para cada normalización
svm_zscore = SVC(kernel='rbf', gamma='scale')
svm_minmax = SVC(kernel='rbf', gamma='scale')
svm_l1 = SVC(kernel='rbf', gamma='scale')
svm_l2 = SVC(kernel='rbf', gamma='scale')

# Normalizar los datos de entrenamiento con cada normalizador
X_train_zscore = scaler_zscore.fit_transform(X_train)
X_train_minmax = scaler_minmax.fit_transform(X_train)
X_train_l1 = scaler_l1.fit_transform(X_train)
X_train_l2 = scaler_l2.fit_transform(X_train)

# Entrenar los modelos con los datos normalizados
svm_zscore.fit(X_train_zscore, y_train)
svm_minmax.fit(X_train_minmax, y_train)
svm_l1.fit(X_train_l1, y_train)
svm_l2.fit(X_train_l2, y_train)
